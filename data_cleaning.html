Cleaning data with data.table

data.table (by Matt Dowle, Arun Srinivasan et. al. ) is a package designed from the ground up in C++ to make cleaning data fast, easy and incredibly powerful. All of the examples below will use real-life 2014 New York air traffic data, which we'll import in the Data I/O section. Don't forget to load the package with library(data.table) first. Unlike in Stata, you have to re-load a package if you want to use it every time you start a new session.

Introduction to data.table

The data.table package centers around data.tables , which are highly efficient data frames that can be manipulated using the package's syntax.

If we have a data.table called d then we can use and manipulate it by putting arguments into its square brackets, i.e. d[].

The three main components of a data.table operation are i, j, and by, which go in the order d[i, j, by] (note you don't have to specify the latter two if you're not currently using them.)

i, the first component, selects the rows of the data.table that you'll be working with, like how in Stata the if or in command options let you refer to certain rows.

j, the second component, both selects and operates on the columns of the data.table, like how in Stata the keep or drop commands select specific columns of your data, or how generate or replace create or modify columns in your data.

by, the third component, gives the variable(s) designating groups that you'll be doing your calculations within, like how in Stata you can precede a command with bysort.

data.table uses these three simple components more flexibly than Stata does. The upshot is that you can can perform complicated operations in a single line of concise data.table code, which may have required multiple commands in other languages to accomplish. But even if you aren't doing anything fancy, data.table has you covered with a stable set of functions that can be deployed on virtually any data wrangling task.

Finally, data.table is extremely fast. It has long set the standard for in-memory data wrangling benchmarks across a variety of libraries and languages. You will likely see an order(s) of magnitude performance difference as you compare the code chunks below. As a bonus for Stata users, many data.table operations can be done in-place, which means that it will affect only the parts you're actually changing, without wasting memory and time on the parts that aren't being changed. Any time in the below cheat sheet you see a function with the word set in it, or the := operator, that's an in-place operation. Speedy!

Cheat Sheet

Data I/O

Read and write .csv

                  import delimited using "https://raw.githubusercontent.com/Rdatatable/data.table/master/vignettes/flights14.csv", clear 
export delimited using "flightdata.csv", replace
                
                  d = fread('https://raw.githubusercontent.com/Rdatatable/data.table/master/vignettes/flights14.csv')
fwrite(d, 'flightdata.csv')

Read and write .dta

                  * .dta is Stata's native (proprietary) filetype 
use "filename.dta", clear 


save "filename.dta", replace
                
                  # These commands require the `haven` package 
d = haven::read_dta('filename.dta') 
setDT(d) # Or: d = as.data.table(d) 
 
haven::write_dta(d, 'filename.dta')

                  order month day
                
                  setcolorder(d, c('month','day'))

Rename columns

                  * rename (old) (new)

rename (carrier origin) (carrier_code origin_code)

                  # setnames(d, old = ..., new = ...)

names(d) = gsub('arr_', 'arrival_', names(d))

Subset

Subset rows

                  keep in 1/200 
keep if day > 5 & day < 10
keep if inrange(day,5,10)
keepif origin == "LGA"
keep if regex(origin,"LGA") 
keep if inlist(month,3,4,11,12) 
keep if inlist(origin,"JFK","LGA") 
drop if month == 1
                
                  d[1:200] 
d[day > 5 & day < 10] 
d[between(day,5,10)] # Or: d[day %in% 5:10] 
d[origin=='LGA']
d[origin %like% 'LGA'] 
d[month %in% c(3,4,11,12)] 
d[origin %chin% c("JFK","LGA")] # %chin% is a faster %in% for (ch)aracter strings 
d[month!=1]

Subset columns

                  keep month day carrier 



keep year-arr_delay
keep *_delay 

ds, has(type int) 
keep `r(varlist) 

drop origin dest 


ds, has(type string) 
drop `r(varlist)'
                
                  d[, .(month, day, carrier)] 
d[, list(month, day, carrier)] # same as above 
d[, c('month', 'day', 'carrier')] # ditto 

d[, year:arr_delay] 
d[, .SD, .SDcols=patterns('*_delay')] 

# Matches the two lines on the left: 
d[, .SD, .SDcols=is.integer] 

d[, -c('origin', 'dest')]
d[, c('origin', 'dest') := NULL] # drop by ref 

# Matches the two lines on the left:
d[, .SD, .SDcols=!is.character]

                  # Matches the two lines on the left:

Modify

Create new variables

                  gen dist_sq = distance^2 
gen tot_delay = dep_delay + arr_delay 
gen first_letter = substr(origin, 1,1) 
gen flight_path = origin + '_' + dest 

* These next operations don't have a great Stata 
* equivalent, although you could implement a loop
                
                  d[, dist_sq := distance^2] 
d[, tot_delay := dep_delay + arr_delay] 
d[, first_letter := substr(origin,1,1)] 
d[, flight_path := paste(origin, dest, sep='_')] 

# Multiple variables can be created at the same time 
d[, c('dist_sq', 'dist_cu') := .(distance^2, distance^3)] 
d[, ':=' (dist_sq=distance^2, dist_cu=distance^3)] # "functional" equivalent 

# We can also chain back-to-back d[...][...] 
# (this holds for any data.table operation) 
d[, dist_sq := distance^2][, dist_cu := distance*dist_sq)]

Create new variables within groups

                  bysort origin: egen mean_dep_delay = mean(dep_delay) 
bysort origin dest: egen mean_dep_delay2 = mean(dep_delay) 

* Multiple grouped variables (manual demean ex.) 
foreach x of varlist dep_delay arr_delay air_time 
egen mean_`x'=mean(`x'), by(origin) 
gen `x'_dm = `x` - mean_`x' 
drop mean* 

* Some short-cut symbols 
bysort carrier: g rows_per_carrier = _N 
bysortcarrier: g index_within_carrier = _n 
egen origin_index = group(origin)
                
                  d[, mean_dep_delay := mean(dep_delay), by=origin] 
d[, mean_dep_delay2 := mean(dep_delay), by=.(origin, dest)] 

# Multiple grouped variables (manual demean ex.) 
dmcols = c('dep_delay', 'arr_delay', 'air_time') 
d[, paste0(dmcols, '_dm') := lapply(.SD, \(x) x-mean(x)),
  .SDcols = dmcols,
  by=origin] 

# Some short-cut symbols 
d[, rows_per_carrier := .N, by = carrier] 
d[, index_within_carrier := .I, by = carrier] 
d[, origin_index := .GRP, by = origin]

Modify existing variables

                  replace tot_delay = dep_delay + arr_delay 

* Conditional modification 
replace distance = distance + 1 if month==9
replace distance = 0 in 1
                
                  d[, tot_delay := dep_delay + arr_delay] 

# Conditional modification 
d[month==9, distance := distance + 1]
d[1, distance := 0]

Using Booleans & control-flow

                  generate long_flight = air_time>500 & !missing(air_time) 

generate flight_length = "Long" if air_time>500 & !missing(air_time)
replace flight_length = "Short" if missing(flight_length) & !missing(air_time) 


generate flight_length2 = "Long" 
replace flight_length2 = "Med" if air_time<=500  
replace flight_length2 = "Short" if air_time<=120
                
                  d[, long_flight := air_time>500] 

d[, flight_length := fifelse(air_time>500, 'Long', 'Short')] 
# fifelse is like base-R ifelse, but (f)aster! 

# for nested ifelse, easier to use fcase 
d[, flight_length2 := fcase(air_time<=120, 'Short', 
                            air_time<=500, 'Med', 
                            default = 'Long')]
